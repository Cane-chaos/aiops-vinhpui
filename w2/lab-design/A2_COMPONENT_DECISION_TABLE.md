## 2. Tóm tắt Chiến lược Chọn Component

### Nguyên tắc quyết định:
1. **Log** là cost driver lớn nhất ($15,700 = 37% tổng bill) → ưu tiên thay thế bằng Loki + S3 tiered retention.
2. **Metrics + APM** là cost driver thứ hai ($19,400 = 46%) → thay thế bằng VictoriaMetrics + Tempo. Loại bỏ per-host pricing model của Datadog.
3. **Dashboard** consolidate vào Grafana self-hosted → giải quyết pain point #3 (4 UI) và #8 (onboarding 2-3 tuần).
4. **PagerDuty giữ lại** vì incident routing mature, mobile app tốt, monthly billing linh hoạt. Chỉ giảm user count.
5. **OTel Collector** là keystone decision: tạo abstraction layer giữa app và backend, cho phép dual-write, migration an toàn, và zero vendor lock-in về sau.

### Capability GAP cần chấp nhận:

| Capability mất | Severity | Mitigation |
|---|---|---|
| Datadog APM continuous profiling | Medium | Thêm Pyroscope (OSS) nếu cần, hoặc Grafana Cloud Profiles |
| Datadog container/process live view | Low | cAdvisor metrics + Grafana Kubernetes dashboard |
| Splunk full-text index search (fast grep) | High | Structured logging (JSON) + Loki label filtering. Pain point #1 vẫn tồn tại một phần cho unstructured legacy log |
| Datadog Synthetics multi-region | Medium | Blackbox Exporter single-region. Thêm Grafana Cloud Synthetics ($200/tháng) nếu cần multi-region |
| Splunk compliance/audit reports | High | Cần xây report mới trên Grafana + Loki. Hoặc giữ Splunk read-only cho compliance cho đến khi migration report xong |
| Datadog composite monitors | Low | Grafana multi-condition alerts + recording rules |
