# ADR-001: Chuyển Log Platform từ Splunk Cloud + Datadog Logs sang Grafana Loki + S3 Tiered Retention

## Status
**Accepted**

---

## Context

### Vấn đề hiện tại
Log pipeline hiện tại là **cost driver lớn nhất** trong observability stack, chiếm **$15,700/tháng (37.4%)** từ hai nguồn:
- **Splunk Cloud**: $13,900/tháng cho 52 GB/day indexed với 30-day hot retention.
- **Datadog Logs**: $1,800/tháng cho ~1.05B events/tháng indexed (15-day retention).

Hai hệ thống log song song gây ra:
1. **Phân mảnh dữ liệu**: On-call phải tìm log ở 2 nơi khác nhau với 2 query language khác nhau (Splunk SPL vs Datadog Log Explorer).
2. **Chi phí indexing cao**: Cả Splunk lẫn Datadog đều **full-text index** toàn bộ log content → chi phí tỷ lệ thuận với volume.
3. **Query chậm**: Splunk log search >25s khi query >7 ngày (pain point #1).
4. **Index rotation disruption**: Splunk hot tier rotation gây empty search results 5-15 phút (pain point #6).

### Số liệu liên quan
- Log volume: 52 GB/day (Splunk) + ~35M events/day (Datadog) ≈ tổng ~55 GB/day raw.
- Trong 29 incidents lịch sử, **17 incidents (59%)** cần log analysis cho RCA.
- MTTR trung bình cho incidents cần log: 38 phút.
- Splunk contract còn 7 tháng, 90-day notice required.

---

## Decision

Chuyển **toàn bộ** log pipeline sang **Grafana Loki** với **S3 tiered retention**:
- **Hot tier** (7 ngày): Loki chunks trên EBS gp3 → query nhanh (<5s cho structured queries).
- **Warm tier** (30 ngày): S3 Standard → query chậm hơn (10-30s) nhưng chi phí thấp.
- **Cold tier** (90 ngày): S3 Standard-IA → archive, restore khi cần.
- **Archive** (1 năm+): S3 Glacier Deep Archive → compliance/audit.

Sử dụng **OTel Collector** làm lớp collection thống nhất, output dạng **structured JSON logs** với labels chuẩn hóa (service, level, pod, namespace).

---

## Alternatives Considered

### Alt 1: Giữ nguyên Splunk Cloud + Datadog Logs
- **Pro**: Zero migration risk, team đã quen.
- **Con**: $15,700/tháng tiếp tục. Phân mảnh 2 hệ thống log không giải quyết được. Splunk contract auto-renew risk (pain point #10).
- **Rejected vì**: Không đạt mục tiêu giảm 40% chi phí. $15,700 đã chiếm 37% tổng budget.

### Alt 2: Consolidate toàn bộ vào Splunk Cloud (bỏ Datadog Logs)
- **Pro**: Một log platform duy nhất. Splunk query mạnh (SPL full-text search).
- **Con**: Splunk vẫn đắt (~$15,000/tháng ở volume hiện tại). Splunk per-GB pricing không scale tốt. Vendor lock-in sâu hơn.
- **Rejected vì**: Chỉ tiết kiệm ~$1,800 (Datadog Logs), không đủ đạt mục tiêu 40%.

### Alt 3: Chuyển sang OpenSearch (self-hosted)
- **Pro**: Full-text search index mạnh (tương đương Elasticsearch). Query language quen thuộc. Dashboard Kibana.
- **Con**: OpenSearch vẫn full-text index → storage cost cao. Cluster management phức tạp (JVM tuning, shard management, rolling upgrades). Operational burden cao hơn Loki đáng kể.
- **Rejected vì**: OpenSearch cost ~$3,000-5,000/tháng cho 52 GB/day (compute + storage cho indexing). Loki ~$755/tháng. Operational complexity của OpenSearch/Elasticsearch cluster là well-known pain point.

### Alt 4: Chuyển sang ClickHouse cho log storage
- **Pro**: Column-oriented → query analytical rất nhanh. SQL query language quen thuộc.
- **Con**: Không phải CNCF project. Ít tích hợp native với Grafana (cần plugin). Community support nhỏ hơn Loki. ClickHouse cluster management có learning curve.
- **Rejected vì**: Loki tích hợp native với Grafana stack (zero config cho correlation metric↔log↔trace). ClickHouse sẽ là "thêm 1 tool nữa" thay vì consolidate.

### Alt 5 (Chosen): Grafana Loki + S3 tiered retention
- **Pro**: Index-free architecture (chỉ index labels) → chi phí ingest giảm 95%. Native Grafana integration. LogQL ≈ PromQL syntax → team chỉ cần học 1 query language. Object-storage native → tiered retention tự nhiên.
- **Con**: Không có full-text index → grep trên log content chậm khi scan lượng lớn. LogQL khác SPL → cần training. Complex queries (join, stats) yếu hơn Splunk.

---

## Consequences

### Positive Consequences
1. **Giảm chi phí log 95%**: $15,700 → $755/tháng (tiết kiệm $14,945/tháng).
2. **Loại bỏ phân mảnh log**: 2 log systems → 1. On-call chỉ cần 1 query interface.
3. **Giải quyết pain point #6**: Loki không có index rotation → không bao giờ gặp "empty search results mid-incident".
4. **Giải quyết pain point #10**: Loại bỏ Splunk vendor lock-in. Loki data trên S3 dạng chuẩn.
5. **Tiered retention linh hoạt**: Tự kiểm soát hot/warm/cold thay vì phụ thuộc vendor retention plan.
6. **Cardinality cost control**: OTel Collector filter/transform trước khi ingest → kiểm soát volume tại source.

### Negative Consequences
1. **Mất Splunk full-text search performance**: Grep trên unstructured log content trong Loki chậm hơn Splunk (30-60s vs 5-10s cho 7-day window). **Mitigation**: Chuyển sang structured JSON logging → phần lớn query dùng label filter, không cần grep.
2. **Mất SPL query language**: Team quen SPL cần relearn LogQL. **Mitigation**: LogQL giống PromQL, team đã biết PromQL qua Grafana Cloud. Training 1-2 ngày đủ cho basic queries.
3. **Mất Splunk compliance reports**: Security/audit team dùng Splunk cho compliance. **Mitigation**: Giữ Splunk read-only cho archive cũ (7 tháng contract còn lại). Xây report mới trên Grafana + Loki.
4. **Operational burden tăng**: Self-hosted Loki cần monitor, upgrade, backup. **Mitigation**: Loki microservices mode cho phép scale từng component. Grafana self-monitoring dashboards.
5. **Cold log query chậm**: Archive logs trên S3-IA/Glacier query mất 30-60s (warm) đến hàng giờ (Glacier). **Mitigation**: 95% RCA queries dùng hot data (7 ngày). Cold query chỉ cần cho postmortem (chấp nhận chậm).

---

## Cost / Feature / Risk Analysis

| Trục | Đánh giá |
|---|---|
| **Cost** | $15,700 → $755/tháng (**-95.2%**). Đây là savings lớn nhất trong toàn bộ redesign. Breakeven ngay tháng đầu (Datadog Logs monthly billing). Splunk savings bắt đầu sau khi hết contract (7 tháng). |
| **Feature** | Mất full-text index search (Splunk strength). Bù bằng structured logging + label-based filtering. Gain: unified log platform, native Grafana correlation, tiered retention. Net: slight negative cho forensic log investigation, positive cho daily operations. |
| **Risk** | **HIGH** — Đây là quyết định rủi ro nhất. Log là nguồn RCA chính (59% incidents). Nếu Loki query không đáp ứng on-call workflow → MTTR tăng thay vì giảm. POC 3 ngày (A7) bắt buộc trước khi commit. |

---

## Rollback / Exit Plan

1. **Trước migration (tuần 1-4)**: Dual-write qua OTel Collector → cả Loki và Splunk/Datadog nhận cùng dữ liệu. Nếu Loki fail → chỉ tắt Loki exporter, giữ Splunk.
2. **Sau migration (tuần 5-8)**: Splunk giữ read-only cho archive cũ (7 tháng contract còn lại). Nếu cần quay lại → re-enable Splunk ingest trong OTel Collector (1 config change, <5 phút).
3. **Sau 6 tháng**: Nếu Loki không đáp ứng → chuyển sang ClickHouse hoặc Grafana Cloud Logs (SaaS) vì data đã trên S3 → re-ingest vào backend mới. OTel Collector giữ flexibility.
4. **Worst case**: Gia hạn Splunk contract 3 tháng ($41,700) trong khi tìm alternative. Calendar reminder 120 ngày trước renewal để không lặp lại pain point #10.
